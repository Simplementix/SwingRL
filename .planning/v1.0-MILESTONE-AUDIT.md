## Critical Integration Gap

### Model Path Double-Nesting (CRITICAL)

**Files:** `scripts/main.py:250` + `src/swingrl/execution/pipeline.py:305-306`

`main.py` passes `Path(config.paths.models_dir) / "active"` (= `models/active`) as `models_dir` to `ExecutionPipeline`. Then `pipeline.py:305` constructs paths as `self._models_dir / "active" / env_name / algo_name / "model.zip"`, producing `models/active/active/equity/ppo/model.zip`.

Training saves models to `models/active/equity/ppo/model.zip`. Pipeline looks in `models/active/active/equity/ppo/model.zip`. Models are never found.

**Fix:** Remove `/ "active"` from `main.py:250`. Pass `Path(config.paths.models_dir)` directly — pipeline already appends `/active/`.

### Position Reconciliation Not Scheduled (MEDIUM)

`PositionReconciler` in `src/swingrl/execution/reconciliation.py` is fully implemented but orphaned. No APScheduler job in `jobs.py` or `main.py` calls it. Position drift between DB and broker state will accumulate silently.

**Fix:** Add a daily reconciliation job in `jobs.py` (after equity market close) that instantiates `PositionReconciler` and calls `reconcile()`.

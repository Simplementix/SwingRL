    Create `src/swingrl/features/hmm_regime.py` with class `HMMRegimeDetector`:

    1. `__init__(self, environment: Literal["equity", "crypto"], config: SwingRLConfig)`:
       - Set covariance_type: "full" for equity, "diag" for crypto
       - Set window: equity_hmm_window (1260) or crypto_hmm_window (2000) from config.features
       - Store n_iter, n_inits, ridge from config.features
       - self._model: GaussianHMM | None = None (tracks current fitted model)

    2. `compute_hmm_inputs(self, close: pd.Series) -> np.ndarray`:
       - Log-returns: np.log(close / close.shift(1))
       - Realized volatility: log_returns.rolling(20).std()
       - Stack into (n, 2) array, drop NaN rows
       - Return only the windowed subset (last `self.window` rows)

    3. `initial_fit(self, close: pd.Series) -> GaussianHMM`:
       - Compute HMM inputs from close prices
       - Run n_inits random initializations, each with n_iter iterations
       - Keep model with best log-likelihood (model.score(data))
       - Apply label ordering: sort states so means_[0] has higher return (bull)
       - Store as self._model
       - Return fitted model

    4. `cold_start_fit(self, close: pd.Series) -> GaussianHMM`:
       - For < 500 bars: use informed prior initialization
       - means_init: [[+0.001, low_vol], [-0.001, high_vol]] (positive mean/low vol = bull)
       - Single fit (no random inits) with init_params="" and manual parameter setting
       - Apply label ordering

    5. `warm_start_refit(self, close: pd.Series) -> GaussianHMM`:
       - Apply ridge regularization to previous model's covariance per RESEARCH.md Pattern 2
       - For "full": symmetrize + add np.eye * ridge
       - For "diag": clip minimum to ridge
       - Set init_params="" on new model, copy startprob_, transmat_, means_, covars_ from previous
       - Fit on new windowed data
       - Apply label ordering
       - Update self._model

    6. `_ensure_label_order(self, model: GaussianHMM) -> GaussianHMM`:
       - If means_[0, 0] < means_[1, 0] (state 0 has lower return than state 1): swap all parameters
       - Swap rows/columns of startprob_, transmat_, means_, covars_
       - Return model with bull=state0, bear=state1

    7. `predict_proba(self, close: pd.Series) -> np.ndarray`:
       - Compute inputs, call model.predict_proba(inputs)
       - Return (n_samples, 2) where col 0 = P(bull), col 1 = P(bear)

    8. `store_hmm_state(self, db: DatabaseManager, date: date, probs: np.ndarray, log_likelihood: float) -> None`:
       - Write to hmm_state_history table: (environment, date, p_bull, p_bear, log_likelihood, fitted_at=utcnow)

  <action>
    Create `src/swingrl/features/turbulence.py` with class `TurbulenceCalculator`:

    1. `__init__(self, environment: Literal["equity", "crypto"])`:
       - Set mode: equity uses expanding lookback (after 252-bar init), crypto uses rolling 1080-bar

    2. `compute(self, returns: np.ndarray, current_idx: int) -> float`:
       - returns: (n_periods, n_assets) full return history
       - current_idx: which row is "now"
       - Equity: historical_returns = returns[:current_idx] (expanding)
       - Crypto: historical_returns = returns[max(0, current_idx - 1080):current_idx] (rolling)
       - If insufficient history (< 252 equity, < 360 crypto): return np.nan
       - current_returns = returns[current_idx]
       - mean_returns = historical.mean(axis=0)
       - cov_matrix = np.cov(historical.T)
       - inv_cov = np.linalg.pinv(cov_matrix) -- pseudo-inverse for numerical stability (especially crypto BTC/ETH high correlation)
       - diff = current_returns - mean_returns
       - turbulence = sqrt(diff @ inv_cov @ diff)
       - Return float

    3. `compute_series(self, returns: np.ndarray) -> np.ndarray`:
       - Compute turbulence for each bar from warmup to end
       - Return (n_periods,) array with NaN for warmup bars
       - Useful for batch computation during training data prep

    Turbulence is computed on-the-fly, NOT stored in DuckDB per CONTEXT.md decision.

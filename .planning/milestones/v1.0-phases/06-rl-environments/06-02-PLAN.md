    **1. Create src/swingrl/envs/base.py -- BaseTradingEnv(gymnasium.Env):**

    Constructor `__init__(self, features: np.ndarray, prices: np.ndarray, config: SwingRLConfig, environment: Literal["equity", "crypto"], render_mode: str | None = None)`:
    - `features`: pre-loaded (n_steps, obs_dim) float32 array (full observation including portfolio state placeholder)
    - `prices`: pre-loaded (n_steps, n_assets) float32 array of close prices
    - Determine obs_dim, n_assets, episode_bars, transaction_cost_pct, max_position_size, max_drawdown_pct from config + environment literal
    - Set `observation_space = Box(low=-np.inf, high=np.inf, shape=(obs_dim,), dtype=np.float32)`
    - Set `action_space = Box(low=-1.0, high=1.0, shape=(n_assets,), dtype=np.float32)`
    - Create PortfolioSimulator and RollingSharpeReward
    - Store config references for penalty computation
    - metadata = {"render_modes": ["human", "ansi"]}

    `reset(self, seed=None, options=None) -> tuple[np.ndarray, dict]`:
    - Call `super().reset(seed=seed)` for Gymnasium RNG setup
    - Call `_select_start_step()` (overridden by subclasses -- equity returns 0, crypto returns random)
    - Set `_max_step = _current_step + episode_bars`
    - Reset portfolio and reward function
    - Set `_peak_value = initial_amount` and `_prev_value = initial_amount`
    - Build observation: take features[_current_step] but replace the portfolio state portion with actual portfolio state (100% cash at reset)
    - Return (obs.astype(np.float32), info_dict)

    `step(self, action: np.ndarray) -> tuple[np.ndarray, float, bool, bool, dict]`:
    Follow the exact step sequence from research:
    1. Process actions via process_actions(action, current_weights, deadzone)
    2. Execute trades at current prices via portfolio.rebalance()
    3. Advance _current_step += 1
    4. Compute new portfolio value at new prices
    5. Compute daily_return = (new_value - prev_value) / prev_value
    6. Update _peak_value = max(_peak_value, new_value)
    7. Compute reward = sharpe_reward - risk_penalty
    8. Build observation with current portfolio state
    9. terminated = _current_step >= _max_step - 1
    10. truncated = False (always)
    11. Build info dict with portfolio_value, daily_return, transaction_cost, turbulence, step
    12. Return 5-tuple

    `_build_observation(self, portfolio_state: np.ndarray) -> np.ndarray`:
    - Take features[_current_step] as base
    - Replace last N elements (EQUITY_PORTFOLIO=27 or CRYPTO_PORTFOLIO=9) with actual portfolio_state
    - Cast to float32

    `_get_portfolio_state(self, prices: np.ndarray) -> np.ndarray`:
    - Compute: cash_ratio, total_exposure, daily_return (3 fixed)
    - Per-asset: weight, unrealized_pnl_pct, bars_since_trade (3 * n_assets)
    - Track `_last_trade_step: np.ndarray` per asset for bars_since_trade computation

    `_compute_risk_penalty(self, weights: np.ndarray, portfolio_value: float) -> float`:
    - Position concentration: quadratic penalty on weights exceeding max_position_size
    - Drawdown: linear penalty on drawdown exceeding max_drawdown_pct
    - Use coefficients from config.environment

    `_get_turbulence(self) -> float`:
    - Return 0.0 as default (turbulence data can be optionally passed in features)
    - Subclasses can override or turbulence can be stored in a separate array passed to __init__

    `_select_start_step(self) -> int`:
    - Base implementation returns 0 (overridden by crypto for random start)

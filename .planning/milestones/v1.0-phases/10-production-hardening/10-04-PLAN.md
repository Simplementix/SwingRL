    - Test: evaluate_shadow_promotion returns False when Shadow Sharpe < Active Sharpe
    - Test: evaluate_shadow_promotion returns False when Shadow MDD > 120% Active MDD
    - Test: evaluate_shadow_promotion returns False when CB triggered during shadow period
    - Test: On promotion, active model is archived and shadow model becomes active
    - Test: On failure, shadow model is archived and Discord alert sent with comparison stats
  </behavior>
  <action>
    1. Add shadow_trades DDL to DatabaseManager.init_sqlite_schema() in src/swingrl/data/db.py:
       ```sql
       CREATE TABLE IF NOT EXISTS shadow_trades (
           trade_id TEXT PRIMARY KEY,
           timestamp TEXT NOT NULL,
           symbol TEXT NOT NULL,
           side TEXT NOT NULL,
           quantity REAL NOT NULL,
           price REAL NOT NULL,
           commission REAL DEFAULT 0.0,
           slippage REAL DEFAULT 0.0,
           environment TEXT NOT NULL,
           broker TEXT,
           order_type TEXT,
           trade_type TEXT,
           model_version TEXT NOT NULL
       )
       ```

    2. Create src/swingrl/shadow/shadow_runner.py:
       - `run_shadow_inference(ctx: JobContext, env_name: str) -> None`:
         a. Check if shadow model exists in models/shadow/{env_name}/. If not, return silently.
         b. Load shadow model using same loading pattern as ExecutionPipeline (SB3 model load).
         c. Get latest observation from feature pipeline.
         d. Run shadow model predict() to get actions.
         e. Run through SignalInterpreter + PositionSizer to generate hypothetical trades (but do NOT submit to exchange adapter).
         f. Record hypothetical trades in shadow_trades table with model_version from filename.
         g. Entire function wrapped in try/except that logs errors but NEVER raises.
         h. Log shadow_inference_complete with trade count.

    3. Create src/swingrl/shadow/promoter.py:
       - `evaluate_shadow_promotion(config: SwingRLConfig, db: DatabaseManager, env_name: str, lifecycle: ModelLifecycle, alerter: Alerter) -> bool`:
         a. Count shadow trades for env_name. If fewer than eval threshold (10 equity / 30 crypto from config.shadow), return False.
         b. Compute shadow Sharpe from shadow_trades returns.
         c. Compute active Sharpe from trades table for the same period.
         d. Compute shadow MDD from shadow_trades.
         e. Compute active MDD from trades table.
         f. Check no circuit_breaker_events during shadow period.
         g. If all 3 criteria pass AND config.shadow.auto_promote is True: call lifecycle.promote(env_name), send Discord alert with comparison stats, return True.
         h. If criteria fail after evaluation period complete: call lifecycle.archive shadow model (move to archive), send Discord alert with comparison stats, return False.
         i. If evaluation period not yet complete: return False (still collecting data).

    4. Create tests with mocked DB, mocked models, and mocked alerter. Use tmp_path for model directories.

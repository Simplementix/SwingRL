       - `backup_sqlite(config: SwingRLConfig, alerter: Alerter) -> bool`: Uses sqlite3.backup() API (not shutil.copy) to create timestamped backup of trading_ops.db to backup_dir/sqlite/trading_ops_YYYYMMDD_HHMMSS.db.     3. Create src/swingrl/backup/duckdb_backup.py:
       - `backup_duckdb(config: SwingRLConfig, alerter: Alerter) -> bool`: Connects to DuckDB, runs CHECKPOINT to flush WAL, closes connection, then shutil.copy2 to backup_dir/duckdb/market_data_YYYYMMDD_HHMMSS.ddb. Opens the backup, verifies table count (ohlcv_daily, ohlcv_4h must exist) and row counts > 0. Never rotates (duckdb_rotate is always False per user decision). Sends Discord alert on success/failure. Returns True on success.

    4. Create src/swingrl/backup/offsite_sync.py:
       - `offsite_rsync(config: SwingRLConfig, alerter: Alerter) -> bool`: Runs `rsync -avz --progress backup_dir/ offsite_host:offsite_path/` via subprocess. Logs output. Sends Discord alert on success/failure. Returns True on success. Skips if offsite_host is empty (not configured).

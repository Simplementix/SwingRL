  <behavior>
    - ExecutionPipeline.execute_cycle(env_name, dry_run=False) runs full 5-stage pipeline
    - Pipeline loads 3 models (PPO/A2C/SAC) for the environment on first call (lazy loading)
    - Pipeline gets observation from FeaturePipeline using current date/datetime
    - Pipeline normalizes observation via VecNormalize (training=False, norm_reward=False)
    - Pipeline calls model.predict(obs, deterministic=True) for each agent
    - Pipeline loads ensemble weights from DuckDB model_metadata table (stored weights, per RESEARCH.md open question 1)
    - Pipeline calls EnsembleBlender.blend_actions() with weights
    - Pipeline passes blended actions through stages 1-5 sequentially
    - If any stage vetoes (RiskVetoError, CircuitBreakerError), pipeline catches and logs, returns empty list
    - Pipeline checks turbulence before executing (PAPER-20): compute current turbulence, compare vs historical 90th percentile
    - Dry-run mode: runs stages 1-3 (signal, size, validate) but skips stage 4 (submit) and 5 (process). Logs what WOULD happen.
    - Returns list of FillResult for successful fills
    - PositionReconciler.reconcile(env) compares DB positions vs broker positions
    - Reconciler trusts broker as source of truth, auto-corrects DB, inserts adjustment trades
    - Reconciler sends Discord warning alert on any mismatch
  </behavior>
  <action>
    Create `src/swingrl/execution/pipeline.py`:
    - `ExecutionPipeline` class constructor takes: config, db (DatabaseManager), feature_pipeline (FeaturePipeline), alerter (Alerter), models_dir (Path)
    - Lazy-initialize all components on first execute_cycle call: signal_interpreter, position_sizer, order_validator, adapters (dict[str, ExchangeAdapter]), fill_processor, risk_manager, circuit_breakers, position_tracker
    - `_load_models(env_name) -> dict[str, tuple]`: load PPO, A2C, SAC from models/active/{env}/{algo}/model.zip and vec_normalize.pkl. Cache in self._models dict. Set VecNormalize training=False, norm_reward=False. Use DummyVecEnv wrapper for VecNormalize loading.
    - `_get_ensemble_weights(env_name) -> dict[str, float]`: query DuckDB model_metadata table for latest weights for this env. Columns: model_id, env_name, algo_name, ensemble_weight. Return dict like {"ppo": 0.4, "a2c": 0.3, "sac": 0.3}.
    - `_get_adapter(env_name) -> ExchangeAdapter`: return AlpacaAdapter for "equity", BinanceSimAdapter for "crypto".
    - `execute_cycle(env_name: str, dry_run: bool = False) -> list[FillResult]`:
      1. Check circuit breaker state. If HALTED, log and return [].
      2. Check turbulence (PAPER-20): get current turbulence from FeaturePipeline (compute from recent returns). Compare vs historical 90th percentile. If exceeded, trigger CB, signal liquidation. For now, liquidation means: log the intent, return [] (actual liquidation implementation is a safety measure -- the CB halt prevents further trading).
      3. Get observation: call feature_pipeline.get_observation(env, current_date_str).
      4. Get portfolio state: call position_tracker.get_portfolio_state_array(env). Replace default portfolio state in observation if needed.
      5. Normalize observation via VecNormalize.
      6. Run predict on each model, collect actions.
      7. Load ensemble weights, blend actions.
      8. Get current asset weights from position_tracker.
      9. Stage 1: signal_interpreter.interpret(env, blended_actions, current_weights).
      10. For each TradeSignal that is not "hold":
          a. Get current price from adapter.
          b. Get ATR value from DuckDB features table (latest ATR for symbol).
          c. Stage 2: position_sizer.size(signal, price, atr, portfolio_value).
          d. If None (Kelly <= 0 or risk-capped), skip.
          e. Stage 3: order_validator.validate(sized_order). Catch RiskVetoError -> log, skip.
          f. If dry_run: log "DRY RUN: would submit {order}" and skip stages 4-5.
          g. Stage 4: adapter.submit_order(validated_order).
          h. Stage 5: fill_processor.process(fill).
          i. Append fill to results.
      11. Record portfolio snapshot after all fills.
      12. Return list of fills.
    - Log extensively with structlog at every stage boundary.

    Create `src/swingrl/execution/reconciliation.py`:
    - `PositionReconciler` class with constructor taking: config, db, adapter (for equity), alerter
    - `reconcile(env_name: str) -> list[dict]`:
      1. Get DB positions from PositionTracker.get_positions(env).
      2. Get broker positions from adapter.get_positions() (only for equity -- crypto uses virtual balance).
      3. If env is "crypto", skip (no broker-side state to reconcile in paper mode). Return [].
      4. Compare: for each broker position, check if DB has matching (symbol, env) with same quantity.
      5. For mismatches: compute delta. Insert adjustment trade via FillProcessor.record_adjustment(). Log warning.
      6. For positions in broker but not DB: create new position row.
      7. For positions in DB but not broker: delete DB position (broker is source of truth).
      8. If any mismatches found, send Discord warning alert via alerter.
      9. Return list of adjustments made.

    Write tests:
    - test_pipeline.py: mock ALL external components (FeaturePipeline, models, adapters, DB). Test full execute_cycle flow end-to-end. Test dry_run mode skips submission. Test CB halt returns empty. Test turbulence crash protection. Test multiple signals in one cycle.
    - test_reconciliation.py: test no mismatch returns empty. Test quantity mismatch creates adjustment. Test missing DB position creates it. Test extra DB position deletes it. Test crypto env skips reconciliation.

    Create `scripts/seed_production.py`:
    - Implements Doc 14 SS10.1 7-step DB seeding procedure:
      1. argparse CLI: `--source-dir` (path to M1 Mac DB files), `--target-dir` (homelab db/ directory, default db/)
      2. Validate source files exist: market_data.ddb, trading_ops.db
      3. Compute SHA256 checksums of source files.
      4. Copy files to target directory using shutil.copy2.
      5. Compute SHA256 of copied files, verify match.
      6. Open each DB and run integrity checks: DuckDB PRAGMA integrity_check equivalent (SELECT count(*) from key tables), SQLite PRAGMA integrity_check.
      7. Print summary: files copied, sizes, checksums verified, integrity checks passed.
    - Note: SCP transfer from M1 Mac to homelab is manual (ssh/scp). This script handles the receiving end: verify + validate after transfer.

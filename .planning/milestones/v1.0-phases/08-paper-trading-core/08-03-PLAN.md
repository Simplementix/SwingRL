    Create `src/swingrl/execution/adapters/binance_sim.py`:
    - `BinanceSimAdapter` class implementing ExchangeAdapter
    - Constructor takes `SwingRLConfig`, `DatabaseManager`, optional `Alerter`
    - `BINANCE_US_BASE = "https://api.binance.us"` as class constant
    - `_get_mid_price(symbol: str) -> tuple[float, float, float]`: GET `/api/v3/depth?symbol={symbol}&limit=5`. Parse bids[0][0] and asks[0][0]. Compute mid = (bid + ask) / 2. Check spread: if (ask - bid) / mid > 0.005, log warning. Return (mid, bid, ask). Use `requests.get()` with timeout=10.
    - `submit_order(order: ValidatedOrder) -> FillResult`:
      1. Map symbol from config format (e.g., "BTCUSDT") to Binance API format (same -- Binance uses BTCUSDT).
      2. Get mid-price via `_get_mid_price()`.
      3. Apply slippage: buy => mid * (1 + slippage), sell => mid * (1 - slippage).
      4. Commission: order.sized_order.dollar_amount * 0.001 (0.10% per side).
      5. Generate trade_id via `str(uuid.uuid4())`.
      6. Return FillResult with fill_price, commission, slippage amount.
      7. Log with structlog: symbol, side, fill_price, mid_price, slippage, commission.
    - `get_positions()`: query SQLite positions table for environment="crypto".
    - `get_current_price(symbol)`: call `_get_mid_price(symbol)[0]`.
    - `cancel_order(order_id)`: no-op for simulated fills (return True, log info).
    - Retry on `_get_mid_price`: 3 attempts with backoff for requests exceptions.

    - `process(fill: FillResult) -> None`:

If sell reduces quantity to 0, DELETE the position row.
      3. If new position (buy with no existing row), INSERT with cost_basis=fill_price.
      4. Log fill processing with structlog.
    - `record_adjustment(symbol, env, quantity_delta, price, reason)`: for reconciliation adjustments -- inserts trade with trade_type="adjustment".

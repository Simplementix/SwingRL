  <behavior>
    - SignalInterpreter.interpret(env, blended_actions, current_weights) returns list[TradeSignal]
    - Actions within +/-0.02 of current weight are "hold" (deadzone from TRAIN-09)
    - Action > current_weight + 0.02 is "buy", action < current_weight - 0.02 is "sell"
    - raw_weight is the target weight from ensemble output
    - PositionSizer.size(signal, current_price, atr_value, portfolio_value) returns SizedOrder
    - Kelly formula: K = win_rate - (1 - win_rate) / reward_risk_ratio, then quarter: K * 0.25
    - If Kelly <= 0, skip trade (return None)
    - Max risk per trade: 2% of portfolio_value
    - Stop-loss: current_price - 2 * ATR for buy, current_price + 2 * ATR for sell
    - Take-profit: R:R ratio from stop distance (default 2:1)
    - Dollar amount = min(kelly_sized, 2% risk cap) * portfolio_value
    - Crypto floor: if env is crypto and dollar_amount < min_order_usd ($10), set to $10.00
    - After floor adjustment, re-check: if $10 > max_position_size * portfolio_value, return None (skip trade per Pitfall 4)
    - Quantity = dollar_amount / current_price
    - For equity: use notional (dollar) amounts, not share quantity (Alpaca supports notional orders)
  </behavior>
  <action>
    Create `src/swingrl/execution/signal_interpreter.py`:
    - `SignalInterpreter` class with constructor taking `SwingRLConfig`
    - `interpret(env_name, blended_actions: np.ndarray, current_weights: np.ndarray) -> list[TradeSignal]`:
      Get symbols from config (equity: config.equity.symbols, crypto: config.crypto.symbols).
      For each symbol index, compare blended_actions[i] vs current_weights[i].
      Apply +/-0.02 deadzone. Create TradeSignal for non-hold actions.
      Log all signals with structlog (env, symbol count, hold count, trade count).

    Create `src/swingrl/execution/position_sizer.py`:
    - `PositionSizer` class with constructor taking `SwingRLConfig`
    - Store default trading stats: win_rate=0.50, avg_win=0.03, avg_loss=0.02 (conservative defaults for paper phase). These will be updated from backtest results in later phases.
    - `size(signal: TradeSignal, current_price: float, atr_value: float, portfolio_value: float) -> SizedOrder | None`:
      1. Compute Kelly: K = win_rate - (1-win_rate) / (avg_win/avg_loss). Quarter-Kelly: K * 0.25.
      2. If K <= 0, log and return None.
      3. Risk-cap: min(quarter_kelly, 0.02).
      4. Dollar amount: risk_capped_pct * portfolio_value.
      5. Stop-loss: buy => price - 2*ATR, sell => price + 2*ATR.
      6. Take-profit: buy => price + 4*ATR (2:1 R:R), sell => price - 4*ATR.
      7. Crypto floor: if env == "crypto", dollar_amount = max(dollar_amount, config.crypto.min_order_usd).
      8. Post-floor risk check: if dollar_amount > max_position_size * portfolio_value, log warning and return None.
      9. Quantity: dollar_amount / current_price.
      10. Return SizedOrder.
    - Log all sizing decisions with structlog.

    Write comprehensive tests:
    - test_signal_interpreter.py: deadzone filtering, buy/sell classification, correct symbol mapping, all-hold returns empty list
    - test_position_sizer.py: kelly formula correctness, quarter-kelly scaling, 2% risk cap, ATR stop/TP levels, crypto $10 floor, post-floor risk rejection, negative kelly returns None

    - OrderValidator.validate(order: SizedOrder) returns ValidatedOrder or raises RiskVetoError
    - Cost gate: estimate round-trip costs and reject if > 2.0% of order value
    - Equity RT cost estimate: ~0.06% (SEC fee + TAF, no commission on Alpaca)
    - Crypto RT cost estimate: 0.22% (Binance.US 0.10% maker + 0.10% taker + slippage)
    - Cost gate rejection logged to risk_decisions table
    - Delegates to RiskManager.evaluate() for risk checks (position size, exposure, drawdown, daily loss, global)
    - Returns ValidatedOrder with list of passed check names
  </behavior>
  <action>
    Create `src/swingrl/execution/order_validator.py`:
    - `OrderValidator` class with constructor taking `SwingRLConfig`, `RiskManager`
    - Define cost rates as class constants: EQUITY_RT_COST_PCT = 0.0006, CRYPTO_RT_COST_PCT = 0.0022
    - `validate(order: SizedOrder) -> ValidatedOrder`:
      1. Estimate RT cost: order.dollar_amount * cost_rate based on environment
      2. Compute cost_pct = rt_cost / order.dollar_amount
      3. If cost_pct > 0.02 (2%), raise RiskVetoError("cost_gate: RT cost {cost_pct:.4%} exceeds 2.0% threshold")
      4. Call self._risk_manager.evaluate(order) -- may raise RiskVetoError or CircuitBreakerError
      5. Return ValidatedOrder(sized_order=order, risk_checks_passed=["cost_gate", "position_size", "exposure", "drawdown", "daily_loss", "global_aggregator"])
    - Log validation result with structlog.

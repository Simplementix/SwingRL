   - Base: python:3.11-slim
   - Copy uv binary from ghcr.io/astral-sh/uv:0.4.30 (or latest 0.4.x/0.5.x stable)
   - WORKDIR /app
   - Two-stage uv sync for Docker layer caching:
     Stage 1: Mount bind pyproject.toml + uv.lock, run `uv sync --locked --no-dev --no-install-project`
     Stage 2: COPY . /app, run `uv sync --locked --no-dev`
   - For CI/test usage, also create a dev target or use --group dev when running tests. Consider a multi-stage approach:
     - Production stage: --no-dev
     - Dev/test stage: inherits from production, adds `uv sync --locked --group dev`

   - USER trader
   - ENTRYPOINT: `["uv", "run", "python", "-m", "swingrl"]`

   IMPORTANT: The Dockerfile must support running tests in CI. Use a build target or ensure dev deps are available when running `docker compose run --rm swingrl uv run pytest`.

   ```yaml

     swingrl:
       build: .
       mem_limit: 2.5g
       cpus: 1.0
       env_file: .env
       environment:
         - TZ=America/New_York
       volumes:
         - ./data:/app/data
         - ./db:/app/db
         - ./models:/app/models
         - ./logs:/app/logs
       restart: unless-stopped
   ```

<task type="auto">
  <name>Task 2: Create ci-homelab.sh with full quality gate and validate locally</name>

  <action>
1. Create scripts/ci-homelab.sh with the full quality gate per user decision (Docker build + pytest + ruff/black/mypy checks inside container):

   ```bash
   #!/usr/bin/env bash
   set -euo pipefail

   REPO_DIR="${REPO_DIR:-$HOME/swingrl}"
   NO_CACHE="${1:-}"

   cd "$REPO_DIR"

   echo "=== [1/5] Git pull ==="
   git pull --ff-only

   echo "=== [2/5] Docker build ==="
   if [[ "$NO_CACHE" == "--no-cache" ]]; then
       docker compose build --no-cache
   else
       docker compose build
   fi

   echo "=== [3/5] Run tests ==="
   docker compose run --rm swingrl uv run pytest tests/ -v

   echo "=== [4/5] Lint + type check ==="
   docker compose run --rm swingrl uv run sh -c 'ruff check . && ruff format --check . && mypy src/'

   echo "=== [5/5] Cleanup ==="
   docker compose down
   docker image prune -f

   echo "=== CI PASSED ==="
   ```

   NOTE on black vs ruff format: Per user decision "ruff/black/mypy checks", use `ruff format --check .` as the formatting check since ruff's formatter is a drop-in black replacement and black is not a separate dependency. This satisfies the intent of the user's decision (formatting + linting + type checking) while keeping the tool chain lean. Stage [4/5] runs all three quality checks: ruff check (linting), ruff format --check (formatting/black equivalent), mypy (type checking).

2. Make executable: `chmod +x scripts/ci-homelab.sh`

3. Validate the script locally first (not via SSH — just run it directly on M1 Mac to confirm the 5 stages work):
   - Set REPO_DIR to current directory
   - Run: `REPO_DIR=$(pwd) bash scripts/ci-homelab.sh`
   - Confirm all 5 stages complete and "CI PASSED" is printed

4. Clean up after local validation: `docker compose down`

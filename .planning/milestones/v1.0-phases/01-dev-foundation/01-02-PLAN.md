CRITICAL ORDERING: detect-secrets baseline MUST be generated and committed BEFORE pre-commit install. Follow this exact sequence:

1. Create .pre-commit-config.yaml using the template from 01-RESEARCH.md "Pattern 3: Pre-commit Configuration" with these hooks:

      - ruff hook with args: [--fix]
      - ruff-format hook (do NOT also include standalone black hook — ruff-format replaces black in pre-commit to avoid formatting conflicts; black config in pyproject.toml is retained for direct CLI use)

   b) mypy (pre-commit/mirrors-mypy):

      - args: [--config-file=pyproject.toml]
      - Pass src/ files only to avoid checking test files with strict rules

   c) bandit (PyCQA/bandit):
      - args: ["-c", "pyproject.toml"]
      - additional_dependencies: ["bandit[toml]"] — REQUIRED for pyproject.toml config reading
      - exclude: tests/ (bandit handles this via pyproject.toml config too, but belt-and-suspenders)

   d) detect-secrets (Yelp/detect-secrets):
      - args: ["--baseline", ".secrets.baseline"]

   ```
   uv run detect-secrets scan > .secrets.baseline
   ```

   ```
   git add .secrets.baseline
   git commit -m "chore: add detect-secrets baseline"
   ```

   ```
   uv run pre-commit install
   ```

   ```
   uv run pre-commit run --all-files
   ```

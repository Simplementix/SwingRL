## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] DuckDB INTEGER PRIMARY KEY requires explicit sequence for auto-increment**
- **Found during:** Task 2 GREEN phase — `test_telemetry_tables_insertable` failed with `NOT NULL constraint failed: training_epochs.id`
- **Issue:** DuckDB's `INTEGER PRIMARY KEY` does not auto-increment unlike SQLite. Inserting without `id` fails.
- **Fix:** Added `CREATE SEQUENCE IF NOT EXISTS` statements and `DEFAULT nextval('seq')` to each table's id column. Sequences are idempotent.
- **Files modified:** `src/swingrl/features/schema.py`
- **Commit:** c3d9d3b (included in same task commit)

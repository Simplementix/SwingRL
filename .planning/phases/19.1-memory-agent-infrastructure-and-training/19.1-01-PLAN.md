**services/memory/db.py:**
- SQLite connection to path from MEMORY_DB_PATH env var (default: /app/db/memory.db)
- init_db() creates tables: memories (id, text, source, created_at, archived), consolidations (id, pattern_text, source_count, created_at, conflicting_with FK), consolidation_quality (id, attempt_count, accepted, rejected_reason, created_at)
- Indexes on memories(source) and memories(created_at)
- Helper functions: insert_memory(text, source) -> int, get_memories(source=None, limit=100, since=None, archived=False) -> list[dict], insert_consolidation(pattern_text, source_count, conflicting_with=None) -> int, get_consolidations(limit=50) -> list[dict], log_consolidation_quality(attempt_count, accepted, rejected_reason=None) -> None

**services/memory/auth.py:**
- FastAPI APIKeyHeader Security dependency per RESEARCH.md Pattern 2
- verify_api_key validates X-API-Key header against MEMORY_API_KEY env var
- Raises HTTP 401 if missing/invalid
- /health endpoint is exempt (applied per-router, not globally)

**services/memory/agents/ingest.py:**
- IngestAgent class with store(text: str, source: str) -> int method
- Calls db.insert_memory(), returns row ID
- Wraps memory text with XML delimiters for prompt safety: `<memory>{text}</memory>`

**services/memory/agents/consolidate.py:**
- ConsolidateAgent class with run() async method (scheduled every 30 min)
- Queries recent unarchived memories from SQLite (window: last 50 or since last consolidation)
- Builds system prompt: domain-specific SwingRL context (capital preservation, asset classes, regime types, metric definitions) as hardcoded Python string constant
- Calls Ollama /api/chat (qwen3:14b) via httpx async with format= JSON schema for structured output
- Schema: {pattern_text: str, affected_algos: list[str], affected_envs: list[str], actionable_implication: str, confidence: float}
- Validates response has all required fields; rejected if missing. Retry once with JSON mode fallback, then discard
- Logs quality to consolidation_quality table (attempt_count, accepted/rejected, reason)
- Conflict detection: compares new pattern against existing consolidations; flags contradictions with conflicting_with FK
- Archives source memories after successful consolidation (set archived=1)
- Tool calling for qwen3:14b as first attempt; fallback to format= JSON schema

**services/memory/agents/query.py:**
- QueryAgent class with two async methods:
  - advise_run_config(query: str) -> dict: Builds prompt from query + recent consolidations + raw memories. Calls Ollama /api/chat with structured output schema matching RunConfigResponse fields. Returns clamped defaults on any failure.
  - advise_epoch(query: str) -> dict: Similar but returns reward weight adjustments + stop_training bool + rationale
- System prompt: hardcoded Python string constant with SwingRL training context (algorithms, hyperparameter ranges from bounds.py, reward weight semantics)
- Uses qwen3:14b for both endpoints
- XML/delimiter wrapping around all memory text inserted into prompts
- All Ollama calls via httpx async with configurable timeout from OLLAMA_TIMEOUT env var (default 30s)

**services/memory/routers/core.py:**
- POST /ingest (auth required): accepts {text: str, source: str}, calls IngestAgent.store(), returns {id: int, status: "ok"}
- POST /consolidate (auth required): triggers ConsolidateAgent.run(), returns {status: "ok", consolidated: int}
- GET /health (NO auth): returns {status: "healthy", ollama: bool, db: bool}. Checks SQLite accessible + Ollama /api/tags reachable

**services/memory/routers/training.py:**
- POST /training/run_config (auth required): accepts {query: str}, calls QueryAgent.advise_run_config(), returns RunConfigResponse (learning_rate, entropy_coeff, clip_range, n_epochs, batch_size, gamma, reward_weights, rationale)
- POST /training/epoch_advice (auth required): accepts {query: str}, calls QueryAgent.advise_epoch(), returns EpochAdviceResponse (reward_weights: dict, stop_training: bool, rationale: str)

**services/memory/routers/debug.py:**
- GET /debug/memories (auth required): query params source, limit (default 100), since (ISO datetime). Read-only
- GET /debug/consolidations (auth required): query param limit (default 50). Read-only

**services/memory/main.py:**
- FastAPI app with lifespan context manager
- Lifespan startup: init_db(), _wait_for_ollama() (block until Ollama /api/tags returns 200, poll every 5s, timeout 300s per RESEARCH.md Pattern 4), create ConsolidateAgent, start AsyncIOScheduler with 30-min consolidation job (interval configurable via CONSOLIDATE_INTERVAL_MIN env var)
- Lifespan shutdown: scheduler.shutdown()
- Register routers: core (no prefix), training (prefix=/training), debug (prefix=/debug)
- Configure structlog with JSON output

IMPORTANT: services/memory/ is SELF-CONTAINED. It must NOT import anything from src/swingrl/. All config comes from env vars (MEMORY_API_KEY, MEMORY_DB_PATH, OLLAMA_URL, OLLAMA_TIMEOUT, CONSOLIDATE_INTERVAL_MIN). This is per RESEARCH.md Pitfall 4.

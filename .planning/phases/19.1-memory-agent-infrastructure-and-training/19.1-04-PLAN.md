**Homelab deployment steps (executed via SSH):**

1. Push branch to origin and pull on homelab:
```bash
ssh homelab "cd ~/swingrl && git fetch origin && git checkout {branch} && git pull origin {branch}"
```

2. Run CI on homelab:
```bash
ssh homelab "cd ~/swingrl && bash scripts/ci-homelab.sh --no-cache"
```

3. Generate MEMORY_API_KEY and add to .env:
```bash
ssh homelab "cd ~/swingrl && echo 'MEMORY_API_KEY='$(openssl rand -hex 16) >> .env"
```

4. Pull Ollama models (first time only — models persist in named volume):
```bash
ssh homelab "cd ~/swingrl && docker compose -f docker-compose.prod.yml up -d swingrl-ollama && sleep 30 && bash scripts/setup_ollama.sh"
```

5. Start full stack (Ollama -> memory -> swingrl):
```bash
ssh homelab "cd ~/swingrl && docker compose -f docker-compose.prod.yml up -d"
```

6. Verify memory service health:
```bash
ssh homelab "curl -s http://localhost:8889/health | python3 -m json.tool"
```

7. Seed memory with backtest data (re-run against live service per CONTEXT.md):
```bash
ssh homelab "cd ~/swingrl && docker compose -f docker-compose.prod.yml exec swingrl uv run python scripts/seed_memory_from_backtest.py"
```

8. Run validate_memory.py to confirm DuckDB schema + memory service connectivity:
```bash
ssh homelab "cd ~/swingrl && docker compose -f docker-compose.prod.yml exec swingrl uv run python scripts/validate_memory.py --memory-url http://swingrl-memory:8889"
```

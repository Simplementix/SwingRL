## Artifacts

### scripts/train_pipeline.py
New functions added at module level:

- `save_training_state(state, path)`: Atomically writes `data/training_state.json` via `.tmp` rename. Survives power outages.
- `load_training_state(path)`: Loads state or returns `{"completed_iterations": [], "current_iteration": 0}` default.
- `select_best_per_algo_env(state)`: Scans all `iteration_N_result` entries in state, picks highest Sortino per algo x env (Calmar tiebreak). Falls back to ensemble Sharpe when per-algo Sortino is missing.
- `deploy_best_models(winners, models_dir)`: `shutil.copy2` from `models/iterations/iter_N/{env}/{algo}/` to `models/active/{env}/{algo}/`. Creates directories. Warns (non-fatal) on missing source files.
- `write_comparison_report(state, report_path, winners)`: Writes `data/training_comparison.json` with `iteration_N` keys, `memory_enabled` flag per iteration, `winners` section.
- `check_memory_service_health(base_url)`: `GET /health` via urllib, returns True on HTTP 200.
- `wait_for_memory_service(base_url)`: Indefinite polling at 30s intervals, logs warnings.
- `run_all_iterations(base_config, iterations, ...)`: Orchestrates 0..N iterations. Deep-copies config per iteration. Sets `memory_agent.enabled=False` for i=0, True for i>0. Saves state atomically before/after each iteration. Retries failed env once. Calls `consolidate()` after each memory iteration.

CLI additions:
- `--iterations N`: 0 = baseline only (default), N > 0 = baseline + N memory iterations
- `--state-path`: Path to training state JSON (default: `data/training_state.json`)
- `--comparison-path`: Path to comparison report JSON (default: `data/training_comparison.json`)
- `main()`: Delegates to `run_all_iterations()` when `--iterations > 0`

### scripts/run_iterations.sh
Operator convenience script: `exec python scripts/train_pipeline.py --env all --iterations $ITERATIONS --force "$@"` (ITERATIONS defaults to 5). chmod +x.

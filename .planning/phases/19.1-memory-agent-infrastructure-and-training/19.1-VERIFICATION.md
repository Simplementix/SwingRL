### Gaps Summary

**Gap 1: Test isolation failure (sys.path contamination) — Blocker**

When pytest runs the full test suite, `tests/training/test_train_pipeline.py` inserts `scripts/` onto `sys.path` at line 33 to import `train_pipeline`. This path is never removed. When `tests/test_memory_service.py` subsequently executes `import main` (at lines 67, 594, 608), Python resolves `main` to `scripts/main.py` (the SwingRL production APScheduler entrypoint) instead of `services/memory/main.py`. The result is `ModuleNotFoundError: No module named 'agents.consolidate'` because scripts/main.py does not import memory agents.

The fix is straightforward: replace the raw `sys.path.insert` in `test_train_pipeline.py` with `importlib.util.spec_from_file_location` (no sys.path mutation needed) or add a fixture that removes `scripts/` from sys.path after the import. This is a single-file change.

**Full suite outcome:** 4 failed, 27 errors, 1028 passed when all tests run together. The 1028 passing tests confirm all other functionality is correct; only the memory service tests in combination are broken.

**Gap 2: Homelab training run not confirmable from local repo**

The phase goal requires 6 training iterations to have actually executed, producing `data/training_comparison.json` and deploying models to `models/active/`. These artifacts are gitignored by design (models and training data should not be in source control). The SUMMARY for Plan 04 states "Task 3: Checkpoint approved — operator verification" but the checkpoint in the plan only required "Training launched on homelab; at least iteration 0 (baseline) has started or completed" for completion.

The infrastructure to run the iterations is completely wired and correct. Whether the iterations have actually completed on homelab requires human confirmation (verification steps 1-4 above).

    - Fields: Consecutive Cash Cycles, Last Non-Trivial Action, Current Regime, Turbulence Level
    - Description: explanation of what stuck agent means

    `build_circuit_breaker_embed(environment: str, trigger_type: str, current_drawdown: float, threshold: float, cooldown_end: str | None = None) -> dict`:
    - Title: "Circuit Breaker: {env.title()}"
    - Color: 0xFF0000 (red/critical)
    - Fields: Trigger Type, Current Drawdown, Threshold, Cooldown Until

    **Stuck agent detection (src/swingrl/monitoring/stuck_agent.py):**

    `check_stuck_agents(db: DatabaseManager) -> list[dict]`:
    - For each env in ("equity", "crypto"), set threshold (10, 30)
    - Query: SELECT cash_balance, total_value FROM portfolio_snapshots WHERE environment = ? ORDER BY timestamp DESC LIMIT ?
    - If len(rows) >= threshold AND all rows have abs(cash_balance - total_value) < 0.01 (all cash):
      - Query last non-cash snapshot date for diagnostics
      - Append to alerts list: {"environment": env, "consecutive_cash_cycles": len(rows), "last_action_date": last_action_date}
    - Return alerts list

    Note: For equity, the threshold of 10 means 10 consecutive trading day snapshots (the query gets latest N snapshots; since equity only creates snapshots on trading days, this naturally counts trading days). For crypto, snapshots are created every 4H cycle, so 30 snapshots = 30 cycles.

    Write tests first. For embeds, verify dict structure, required fields, color values. For stuck agent, use in-memory SQLite with test data inserted directly.

    **scripts/main.py:**

    Single-file production entrypoint. Structure:

    ```
    1. Parse args (--config, default config/swingrl.yaml)
    2. Load config via load_config(args.config)
    3. Configure logging via configure_logging()
    4. Create DatabaseManager
    5. Init emergency flags table
    6. Create ExecutionPipeline
    7. Create Alerter (with two-webhook URLs from config.alerting)
    8. Init job context (passes config, db, pipeline, alerter to jobs module)
    9. Create BackgroundScheduler with:
       - SQLAlchemyJobStore(url=f"sqlite:///{config.scheduler.apscheduler_db_path}")
       - ThreadPoolExecutor(max_workers=config.scheduler.max_workers)
       - job_defaults: coalesce=True, max_instances=1, misfire_grace_time=config.scheduler.misfire_grace_time
    10. Register all cron jobs:
        - equity_cycle: cron hour=16, minute=15, timezone="America/New_York", id="equity_cycle", replace_existing=True
        - crypto_cycle: cron hour="0,4,8,12,16,20", minute=5, timezone="UTC", id="crypto_cycle", replace_existing=True
        - daily_summary_job: cron hour=18, minute=0, timezone="America/New_York", id="daily_summary", replace_existing=True
        - stuck_agent_check_job: cron hour=17, minute=30, timezone="America/New_York", id="stuck_agent_check", replace_existing=True
        - weekly_fundamentals_job: cron day_of_week="sun", hour=18, minute=0, timezone="America/New_York", id="weekly_fundamentals", replace_existing=True
        - monthly_macro_job: cron day=1, hour=18, minute=0, timezone="America/New_York", id="monthly_macro", replace_existing=True
    11. Start scheduler
    12. Start stop-price polling daemon thread
    13. Register SIGTERM/SIGINT handler: log shutdown, scheduler.shutdown(wait=False)
    14. Block: threading.Event().wait() (works on both Mac and Linux, unlike signal.pause which is Unix-only in some edge cases)
    ```

    Log startup info: config path, trading mode, job count, scheduler jobstore path.

    **src/swingrl/scheduler/stop_polling.py:**

    `start_stop_polling_thread(config: SwingRLConfig, db: DatabaseManager) -> threading.Thread`:
    - Creates and starts a daemon thread running `_poll_stop_prices(config, db)`
    - Returns the thread (for testing/reference, main.py doesn't need to join it)

    `_poll_stop_prices(config: SwingRLConfig, db: DatabaseManager) -> None`:
    - Infinite loop:
      1. if is_halted(db): log and sleep(60), continue
      2. Query open positions from position_tracker (SQLite)
      3. For crypto positions only: fetch current price from Binance.US REST API (httpx.get)
      4. Compare current price vs stop_loss and take_profit from the position record
      5. If stop/TP triggered: log the trigger, send critical alert (import Alerter), mark position
      6. sleep(60)
    - Entire loop body wrapped in try/except: log.exception on any error, sleep(60), continue (never crash the thread)
    - Thread daemon=True set by caller

    Note: The position_tracker table may not have stop_loss/take_profit columns yet. If so, query from trade_log or order records. Use try/except for missing columns and log a warning. This is a best-effort daemon -- Phase 10 hardening will refine it.

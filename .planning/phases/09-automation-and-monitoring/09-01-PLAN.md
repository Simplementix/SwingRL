    Create a `JobContext` dataclass or simple container that holds references needed by all jobs:
    - config: SwingRLConfig
    - db: DatabaseManager
    - pipeline: ExecutionPipeline
    - alerter: Alerter

    Module-level `_ctx: JobContext | None = None` with `init_job_context(config, db, pipeline, alerter)` to set it. Jobs access `_ctx` (fail fast if None).

    **equity_cycle() -> list[FillResult]:**
    1. Check is_halted(_ctx.db) -- if True, log and return []
    2. try: fills = _ctx.pipeline.execute_cycle("equity")
    3. Post-cycle callbacks: send trade alerts for each fill (call into embeds from Plan 02 -- import guarded), run wash sale scan (import guarded), ping HC
    4. except Exception: log.exception, send critical alert, return []
    5. Return fills

    **crypto_cycle() -> list[FillResult]:**
    Same pattern with "crypto" env. No wash sale scan (equities only). Ping crypto HC URL.

    **daily_summary_job() -> None:**
    1. Check is_halted
    2. Query portfolio_snapshots for latest per-environment snapshot
    3. Build daily summary (delegate to embeds module from Plan 02 -- import guarded with try/except ImportError for graceful degradation during Plan 01 testing)
    4. Send via alerter to daily webhook

    **stuck_agent_check_job() -> None:**
    1. Check is_halted
    2. For each env, query last N portfolio_snapshots (10 equity, 30 crypto)
    3. If all cash_balance ~= total_value, send stuck agent alert

    **weekly_fundamentals_job() -> None:**
    1. Check is_halted

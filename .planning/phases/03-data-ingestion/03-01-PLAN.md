  <behavior>
    - BaseIngestor: ABC with abstract fetch(), validate(), store() methods; concrete run() method orchestrating fetch->validate->store; _store_quarantine() writes quarantined rows to data/quarantine/
    - ParquetStore.upsert(path, new_df, key_col="timestamp"): if file exists, read + concat + dedup on index (keep latest) + sort + write snappy; if not exists, create parent dirs + write
    - ParquetStore.read(path): read Parquet, return DataFrame (empty DF if file missing)
    - DataValidator.validate_rows(df, symbol): vectorized row-level checks (steps 1-7), returns (clean_df, quarantine_df)
      - Step 1: Null check on OHLCV columns
      - Step 2: Price > 0 for open/high/low/close
      - Step 3: Volume >= 0
      - Step 4: OHLC ordering high >= low
      - Step 5: O/C between H/L (0.01% tolerance)
      - Step 6: Price spike > 50% bar-to-bar
      - Step 7: Zero volume on non-holiday (equity only, requires source parameter)
    - DataValidator.validate_batch(df, symbol, source): batch-level checks (steps 8-12)
      - Step 8: Duplicate timestamps - deduplicate, keep last, log warning
      - Step 9: Gap detection - NYSE calendar for equity, continuous 4H for crypto, per-series for FRED
      - Step 10: Stale data - max(timestamp) older than threshold (2 trading days equity, 8H crypto)
      - Step 11: Row count threshold - warn if < 90% of expected
      - Step 12: Cross-source placeholder (skip in Phase 3, log "deferred to Phase 4")
    - DataValidator accepts source type ("equity", "crypto", "fred") to switch gap detection logic
  </behavior>

    4. Create src/swingrl/data/base.py — BaseIngestor ABC per RESEARCH.md Pattern 1. Constructor takes SwingRLConfig. run() orchestrates fetch->validate->store with structlog. _store_quarantine() writes to data/quarantine/{source}_{symbol}_{date}.parquet

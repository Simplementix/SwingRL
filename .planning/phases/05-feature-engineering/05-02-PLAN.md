    - Test: fetch_all(symbols) returns DataFrame with one row per symbol, all 4 metrics + sector column

  <action>
    Create `src/swingrl/features/fundamentals.py` with class `FundamentalFetcher`:

    1. `__init__(self, config: SwingRLConfig)`: Store config. Initialize yfinance as primary, alpha-vantage as fallback. Alpha Vantage API key from `ALPHA_VANTAGE_API_KEY` env var (optional -- fallback disabled if missing).

    2. `fetch_symbol(self, symbol: str) -> dict[str, float | None]`:
       - Primary: `yfinance.Ticker(symbol).info` -> extract trailingPE, earningsQuarterlyGrowth, debtToEquity, dividendYield
       - Handle missing keys gracefully (ETFs may lack fields per RESEARCH.md Pitfall 5)
       - On yfinance failure: try Alpha Vantage `FundamentalData.get_company_overview(symbol)` if API key available
       - Return dict with pe_ratio, earnings_growth, debt_to_equity, dividend_yield (float or None)
       - Log each fetch with structlog (symbol, source, fields_available)

    3. `validate_fundamentals(self, data: dict[str, float | None]) -> dict[str, float | None]`:
       - P/E: must be positive or None (negative -> None)
       - D/E: must be non-negative or None (negative -> None)
       - No >100% quarter-over-quarter change validation (requires historical comparison, done at store time)

    4. `fetch_all(self, symbols: list[str]) -> pd.DataFrame`:
       - Iterate symbols, fetch + validate each. Partial failure resilience (try/except per symbol).
       - Return DataFrame: columns = [symbol, pe_ratio, earnings_growth, debt_to_equity, dividend_yield, sector]
       - Sector from yfinance Ticker.info["sector"] or "Unknown"

    5. `sector_relative_zscore(self, fundamentals: pd.DataFrame) -> pd.DataFrame`:
       - For each of [pe_ratio, earnings_growth, debt_to_equity, dividend_yield]:
         - Group by sector, compute z-score = (value - sector_mean) / sector_std
         - Return column named pe_zscore, earnings_growth (raw), debt_to_equity (raw), dividend_yield (raw)
       - Only P/E gets z-scored (per CONTEXT.md: "sector-relative z-score"). Others are raw ratios.

    6. `store_fundamentals(self, db: DatabaseManager, fundamentals: pd.DataFrame) -> int`:
       - Write to DuckDB fundamentals table using replacement scan pattern (Phase 4 established)
       - Return rows written

    Add `uv add alpha-vantage` as new dependency. Mock yfinance and alpha-vantage in tests (no live API calls).

    Create `src/swingrl/features/macro.py` with class `MacroFeatureAligner`:

    1. `__init__(self, db: DatabaseManager)`: Store db reference.

    2. `_fetch_macro_aligned(self, table: str, date_col: str, symbol: str, start: str, end: str) -> pd.DataFrame`:
       - Execute ASOF JOIN query per RESEARCH.md Pattern 3 (chained ASOF JOINs for all 5 FRED series)
       - Join against ohlcv_daily (equity) or ohlcv_4h (crypto) using release_date for look-ahead prevention
       - Return raw aligned values for all 5 series

    3. `compute_derived_macro(self, raw_macro: pd.DataFrame) -> pd.DataFrame`:
       - vix_zscore: (VIX - 252d_rolling_mean) / 252d_rolling_std
       - yield_curve_spread: raw T10Y2Y value (already a spread)
       - yield_curve_direction: 1 if spread > 0, else 0 (binary)
       - fed_funds_90d_change: current DFF - DFF_90_days_ago (use .shift(90) on business days or approximate)
       - cpi_yoy: year-over-year CPI change (current / 12_months_ago - 1)
       - unemployment_3m_direction: 1 if current < 3_months_ago (improving), else 0

    4. `align_equity(self, symbol: str, start: str, end: str) -> pd.DataFrame`:
       - Fetch + compute. Returns 6 macro features aligned to equity daily bars.
       - Forward-fill any remaining NaN from ASOF JOIN gaps.

    5. `align_crypto(self, symbol: str, start: str, end: str) -> pd.DataFrame`:
       - Same as equity but joins against ohlcv_4h.
       - Crypto forward-fills from last available equity close (macro updates on business days only).
       - Returns 6 macro features aligned to crypto 4H bars.

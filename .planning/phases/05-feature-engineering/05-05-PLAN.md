    1. Create `src/swingrl/features/pipeline.py` with class `FeaturePipeline`:

       `__init__(self, config: SwingRLConfig, db: DatabaseManager)`:
       - Instantiate all feature modules: TechnicalIndicatorCalculator, FundamentalFetcher, MacroFeatureAligner, HMMRegimeDetector (equity + crypto), RollingZScoreNormalizer, TurbulenceCalculator (equity + crypto), ObservationAssembler

       `compute_equity(self, symbols: list[str] | None = None, start: str | None = None, end: str | None = None) -> pd.DataFrame`:
       - For each symbol: read ohlcv_daily from DuckDB, compute technical indicators, compute weekly features
       - Fetch fundamentals (if stale or missing), compute sector z-scores
       - Align macro features via ASOF JOIN
       - Normalize all features via rolling z-score (window=252)
       - Store to features_equity table (using replacement scan pattern)
       - Fit/refit HMM on SPY
       - Return features DataFrame

       `compute_crypto(self, symbols: list[str] | None = None, start: str | None = None, end: str | None = None) -> pd.DataFrame`:
       - Similar flow for crypto: read ohlcv_4h, compute technical + multi-timeframe indicators
       - Align macro (forward-fill from last equity close)
       - Normalize (window=360)
       - Store to features_crypto table
       - Fit/refit HMM on BTC
       - Return features DataFrame

       `get_observation(self, environment: Literal["equity", "crypto"], date_or_datetime: str) -> np.ndarray`:
       - Read stored features from DuckDB for the given date
       - Compute turbulence on-the-fly
       - Get HMM probabilities
       - Assemble observation vector
       - Return (156,) or (45,) array

    2. Create feature A/B comparison infrastructure in `pipeline.py`:

       `compare_features(baseline_metrics: dict, candidate_metrics: dict, threshold: float = 0.05) -> dict`:
       - Compare validation Sharpe: candidate - baseline
       - Accept if improvement >= threshold (0.05)
       - Reject if train Sharpe improves but validation Sharpe decreases (overfitting guard)
       - Return {accepted: bool, sharpe_improvement: float, reason: str}

       This is the infrastructure only -- actual A/B testing requires trained agents (Phase 7+).

    3. Create `scripts/compute_features.py` CLI:

       - Log summary stats (rows computed, NaN count, time elapsed)

       Tier 1 (`_tier1_halt_and_cancel`): set_halt(db, reason, "emergency_stop"). Cancel all open orders via Alpaca TradingClient.cancel_orders() and Binance.US cancel (currently sim -- logs warning that live cancel not available in paper mode). Log tier1_complete. Target: < 1 second.

       - If extended hours: submit limit sells at current bid
       - If closed: submit with time_in_force='opg' (on-open) via Alpaca
       Log tier3_complete with strategy used.

       Tier 4 (`_tier4_verify_and_alert`): Check Alpaca positions (should be 0 or queued). Check Binance.US positions (should be 0). Build Discord critical alert embed with: reason, timestamp, tier results, remaining positions (if any). Send via alerter.send_alert("critical", ...). Log tier4_complete.

       `check_automated_triggers(config, db) -> list[str]`:
       Returns list of trigger reasons (empty if no triggers active):
       1. VIX > 40 AND global CB within 2% of firing: Query latest VIX from features, check combined drawdown > -13% (CB fires at -15%)

  <name>Task 1: Run training pipeline on homelab</name>
  <files>scripts/train_pipeline.py</files>
  <action>
SSH into the homelab and execute the training pipeline. Steps:

1. Ensure the feature branch is checked out and up to date on homelab:
   ```
   ssh homelab "cd ~/swingrl && git fetch origin && git checkout {branch} && git pull origin {branch}"
   ```

2. Run a dry-run first to validate data loading:
   ```
   ssh homelab "cd ~/swingrl && uv run python scripts/train_pipeline.py --env all --dry-run"
   ```
   This should exit 0 with data shapes logged. If it fails, diagnose and fix before proceeding.

3. Run the full training pipeline (this will take many hours on CPU):
   ```
   ssh homelab "cd ~/swingrl && nohup uv run python scripts/train_pipeline.py --env all --report-path data/training_report.json > logs/training_run.log 2>&1 &"
   ```
   Use nohup so the training survives SSH disconnection. Note the PID for monitoring.

4. Monitor progress periodically:
   ```
   ssh homelab "tail -50 ~/swingrl/logs/training_run.log"
   ```

5. After completion, verify:
   - Exit code 0 (check logs for final status)
   - All 6 model.zip files exist: `ls -la ~/swingrl/models/active/{equity,crypto}/{ppo,a2c,sac}/model.zip`
   - All 6 vec_normalize.pkl files exist alongside model.zip
   - JSON report exists: `cat ~/swingrl/data/training_report.json | python -m json.tool | head -50`
   - Ensemble gate passed for both environments (check report or logs)

6. Pull the training report back to the local repo for commit:
   ```
   scp homelab:~/swingrl/data/training_report.json data/training_report.json
   ```

If training fails or gate does not pass:
- Check logs for which algo/env failed
- If tuning was triggered, check if both rounds were attempted
- If exit code 2 (deployment blocked): the models are not good enough. Review the diagnostic report in logs. This may require investigating data quality or environment configuration — flag as a blocker for Phase 20.
- If exit code 3 (gate failed, no tuning): Sharpe is between 0.5-1.0. Models exist but did not meet the deployment bar. Review if the gate thresholds are reasonable for the data volume.

IMPORTANT: This task runs in the background on the homelab. Do not wait for it synchronously — it could take 12-48 hours on CPU. Use the checkpoint in Task 2 to verify results.

  <action>
    1. Create scripts/train_pipeline.py implementing the full orchestration pipeline:

       **CLI Interface:**
       - argparse with --env equity|crypto|all, --config (default config/swingrl.yaml), --models-dir (default models), --force (re-run even if checkpoints exist)
       - Invocation: `python scripts/train_pipeline.py --env all`

       **Main Flow (per environment):**
       a. Load features and prices using _load_features_prices() from scripts/train.py (import it)
       b. Open DuckDB connection (per-env, not held for entire pipeline -- per RESEARCH.md pitfall)
       c. For each algo in [ppo, a2c, sac]:
          - Check checkpoint: models/active/{env}/{algo}/model.zip exists -> skip
          - Run walk-forward validation: backtester.run(env, algo, FULL features/prices, models_dir, DEFAULT_TIMESTEPS[env])
          - Store fold results
       d. After all 3 algos complete walk-forward:
          - Compute ensemble weights via compute_ensemble_weights_from_wf()
          - Check ensemble gate via check_ensemble_gate()
       e. If gate fails AND baseline Sharpe < 0.5:
          - Run tuning round 1 (PPO only, 4 hyperparams variants from TUNING_GRID)
          - Re-run walk-forward for best PPO variant
          - Recompute ensemble gate
          - If still fails: run tuning round 2 (A2C + SAC variants)
          - If both rounds fail: BLOCK deployment, print diagnostic, write report, exit 1
       f. Train final deployment models on RECENT data only:
          - Use slice_recent() to get recent features/prices
          - decide_final_timesteps() for timestep selection
          - TrainingOrchestrator.train() with hyperparams_override if tuning found better params
          - If memory_agent.meta_training: wrap with MetaTrainingOrchestrator.run()
       g. Verify model deployment: check model.zip + vec_normalize.pkl exist for all 3 algos
       h. Write model metadata to DuckDB
       i. Write JSON report to data/training_report.json with structure from RESEARCH.md

       **JSON Report Structure:**
       ```
       {
         "generated_at": ISO datetime,
         "equity": {
           "walk_forward": { "ppo": [...folds...], "a2c": [...], "sac": [...] },
           "ensemble_weights": {"ppo": 0.45, "a2c": 0.28, "sac": 0.27},
           "ensemble_gate": {"passed": true, "sharpe": 1.2, "mdd": 0.09},
           "tuning_rounds": [],
           "final_training": {"ppo": {"timesteps": 1000000, "wall_clock_s": 3600}, ...},
           "wall_clock_total_s": 14400
         },
         "crypto": { ... }
       }
       ```

       **Console Output:**
       - Per-algo progress: algo started, fold progress, algo complete
       - Environment summary: pass/fail per algo, ensemble metrics, timing
       - Final summary: overall pass/fail, total wall-clock time

       **Key Invariants:**
       - Walk-forward uses FULL history. Final training uses RECENT data only. Never mix these.
       - Ensemble gate and weight computation are PER-ENVIRONMENT. Never aggregate across equity and crypto.
       - Per-fold gate results are informational only -- only ensemble gate blocks deployment.
       - Open/close DuckDB connection per-algo unit (not held for entire pipeline).

       - test_ensemble_weights_from_wf_sharpe: Verify pipeline passes real OOS Sharpe to EnsembleBlender (not placeholder 1.0).

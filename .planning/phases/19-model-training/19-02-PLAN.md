<task type="auto">
  <name>Task 1: Implement train_pipeline.py CLI and orchestration logic</name>
  <files>scripts/train_pipeline.py</files>
  <action>
Extend scripts/train_pipeline.py (which currently has only helper functions from Plan 01) with the full orchestration pipeline. Add:

**CLI (build_parser):**
- `--env equity|crypto|all` (required). "all" runs equity then crypto sequentially.
- `--config` path to config YAML (default: config/swingrl.yaml)
- `--models-dir` root directory for model storage (default: models)
- `--report-path` JSON report output path (default: data/training_report.json)
- `--dry-run` flag to validate data loading and exit

**Pipeline flow (main function) — per environment:**

1. **Data Loading**: Reuse `_load_features_prices` from scripts/train.py (import it). Open DuckDB connection per-env, close after each env completes (per RESEARCH.md recommendation).

2. **Walk-Forward Validation**: For each algo in [ppo, a2c, sac]:
   - Check checkpoint: if models/active/{env}/{algo}/model.zip exists, skip (log checkpoint_found_skipping).
   - Run `WalkForwardBacktester.run()` with FULL features/prices and DEFAULT_TIMESTEPS[env].
   - Store fold results in `wf_results[algo] = fold_results`.
   - Log per-algo timing.

3. **Ensemble Analysis**:
   - Call `check_ensemble_gate(wf_results)` -> (passed, ensemble_sharpe, ensemble_mdd).
   - Call `compute_ensemble_weights_from_wf(config, wf_results, env_name)` -> weights.

4. **Tuning (conditional)**:
   - If `should_trigger_tuning(ensemble_sharpe)`:
     - Round 1: For each PPO variant in TUNING_GRID[1]["ppo"], re-run walk-forward with `hyperparams_override`. Pick best variant by OOS Sharpe. Replace wf_results["ppo"] with best.
     - Recompute gate. If Sharpe >= 0.5, skip Round 2.
     - Round 2: Same for A2C and SAC variants from TUNING_GRID[2]. Pick best per algo.
     - Recompute gate after Round 2.
   - If both rounds fail AND ensemble_sharpe still < 0.5: log error, print diagnostic report, set deployment_blocked=True.

5. **Final Model Training**:
   - If NOT deployment_blocked AND gate passed (Sharpe > 1.0, MDD < 0.15):
     - For each algo: `slice_recent(full_features, full_prices, env_name)` -> recent data.
     - `decide_final_timesteps(env_name, wf_convergence_info)` -> timesteps.
     - `orchestrator.train(env_name, algo, recent_features, recent_prices, timesteps, hyperparams_override=best_override_if_tuned)`.
     - Verify model.zip and vec_normalize.pkl exist in models/active/{env}/{algo}/.
   - If deployment blocked: skip final training, exit with code 2.
   - If gate failed but Sharpe >= 0.5 (not tuning-worthy, but doesn't meet deployment gate): skip deployment, exit with code 3. Print advisory.

6. **Model Metadata**: Write to DuckDB via `_write_model_metadata` with real validation_sharpe and ensemble_weight.

7. **JSON Report**: Write `data/training_report.json` with structure from RESEARCH.md:
   ```
   { generated_at, equity: { walk_forward: {algo: [fold_details]}, ensemble_weights, ensemble_gate, tuning_rounds, final_training, wall_clock_total_s }, crypto: { ... } }
   ```

8. **Console Summary**: Print formatted summary — pass/fail per algo, ensemble Sharpe/MDD, ensemble weights, gate status, total wall-clock time.

**Key design constraints (from CONTEXT.md locked decisions):**
- Walk-forward uses FULL history (full_features, full_prices).
- Final deployed model trains on RECENT data only (slice_recent).
- Never reuse walk-forward VecNormalize for final model.
- Per-fold gate results are informational — only ensemble gate blocks.
- Sequential execution: equity first, then crypto (when --env all).
- No wall-clock time limit.

**Error handling:**
- Wrap all training calls in try/except. On ModelError or unexpected exception, log with structlog, record in report, exit with code 1.
- Use `ModelError` for training failures per CLAUDE.md exception hierarchy.
- Log every phase transition with structlog keyword args (never f-strings).

**Exit codes:** 0=success, 1=error, 2=deployment_blocked (tuning failed), 3=gate_failed (Sharpe 0.5-1.0).

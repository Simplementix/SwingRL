    - CB triggers when drawdown exceeds env threshold (equity 10%, crypto 12%) or daily loss exceeds limit (equity 2%, crypto 3%)
    - Global CB triggers at -15% combined DD or -3% combined daily loss
    - Halt event persists in circuit_breaker_events SQLite table
    - After DB close/reopen, get_state() still returns HALTED if not resumed
    - Cooldown: 5 business days equity (use exchange_calendars NYSE), 3 calendar days crypto
    - Ramp-up: capacity progresses through [0.25, 0.50, 0.75, 1.00] at equal intervals during cooldown
    - get_capacity_fraction() returns current capacity (1.0 when ACTIVE, 0.0 when HALTED, 0.25-1.0 when RAMPING)
    - RiskManager.evaluate(sized_order, env) returns (approved: bool, vetoed_reasons: list[str])
    - RiskManager checks in order: (1) CB state, (2) position size vs max, (3) exposure check, (4) drawdown check, (5) daily loss check, (6) global aggregator
    - Turbulence crash protection: if turbulence > 90th percentile, return liquidation signal and trigger CB
    - All veto decisions logged to risk_decisions table
  </behavior>
  <action>
    Create `src/swingrl/execution/risk/circuit_breaker.py`:
    - `CBState` enum with ACTIVE, HALTED, RAMPING values
    - `CircuitBreaker` class with constructor taking (environment: str, db: DatabaseManager, config: SwingRLConfig)
    - `check_and_update(portfolio_value, high_water_mark, daily_pnl)`: compute drawdown as `1 - portfolio_value / high_water_mark`. Compare against env threshold from config. If breached, call `_trigger()`. Check daily_pnl against daily_loss_limit_pct. Return current CBState.
    - `get_state()`: load from SQLite. If latest event for this env has `resumed_at IS NULL`, state is HALTED (or RAMPING if cooldown partially elapsed). Otherwise ACTIVE.
    - `_trigger(trigger_value, threshold, reason)`: insert row into circuit_breaker_events with UUID event_id, environment, triggered_at=UTC now, trigger_value, threshold, reason. Log with structlog.
    - `resume()`: update latest event's resumed_at to UTC now. Only call when cooldown complete and ramp at 100%.
    - `get_capacity_fraction()`: if ACTIVE return 1.0, if HALTED check if cooldown has started (time since triggered_at). Equity: use `exchange_calendars.get_calendar("XNYS")` to count business days. Crypto: use `timedelta(days=3)`. Divide elapsed into 4 ramp stages. Return 0.0 if still in initial halt, then 0.25/0.50/0.75/1.00 based on stage. Auto-transition to RAMPING when first ramp stage reached. Auto-resume when 1.00 reached.
    - Global circuit breaker: separate class `GlobalCircuitBreaker` that aggregates across environments. Constructor takes list of per-env CircuitBreakers. Checks combined portfolio_value against combined initial capital for -15% DD and combined daily_pnl for -3% daily.

    Create `src/swingrl/execution/risk/risk_manager.py`:
    - `RiskManager` class with constructor taking (config, db, position_tracker, circuit_breakers: dict[str, CircuitBreaker], global_cb: GlobalCircuitBreaker)
    - `evaluate(order: SizedOrder) -> RiskDecision`: run checks in Doc 04 evaluation order:
      1. Circuit breaker state (raise CircuitBreakerError if HALTED; if RAMPING, scale order size by capacity_fraction)
      2. Per-env position size: order.dollar_amount / portfolio_value <= max_position_size
      3. Per-env exposure: current exposure + new position <= 1.0 (fully invested cap)
      4. Per-env drawdown: current drawdown < max_drawdown_pct (if breached, trigger CB)
      5. Per-env daily loss: daily_pnl > -daily_loss_limit_pct * portfolio_value
      6. Global aggregator: combined DD < 15%, combined daily loss < 3%
    - Log every decision to risk_decisions table via `_record_decision()`
    - `check_turbulence(env, turbulence_value, historical_90th_pct) -> bool`: if turbulence > 90th pct, trigger CB for env, return True (signals liquidation needed). Use structlog critical-level log.
    - Raise `RiskVetoError` with descriptive reason when veto occurs.

  <action>
Create `src/swingrl/training/retraining.py` with:

1. **`compute_rolling_sharpe(db, env_name, window_days=30, min_observations=10) -> float | None`**:
   - Query trades table for the last `window_days` calendar days for given environment
   - Aggregate by trading day to get daily P&L series (sum of realized P&L per day)
   - Count non-zero P&L days. If < min_observations, return None
   - Compute annualized Sharpe: mean(daily_returns) / std(daily_returns) * sqrt(annualization_factor)
   - Annualization: sqrt(252) for equity, sqrt(1460) for crypto (6 bars/day * 365.25)
   - Return the Sharpe ratio as float

2. **`check_performance_trigger(db, config, env_name) -> tuple[bool, str]`**:
   - Compute rolling Sharpe via compute_rolling_sharpe()
   - If None (insufficient data), return (False, "insufficient_data")
   - Read baseline_sharpe from retrain_state for this env
   - If baseline_sharpe is None, return (False, "no_baseline")
   - Check relative decline: if (baseline - current) / baseline > config.retraining.sharpe_decline_threshold, return (True, f"performance_degradation (Sharpe dropped {decline_pct:.0%})")
   - Check absolute floor: if current < config.retraining.sharpe_floor, return (True, f"sharpe_below_floor ({current:.2f} < {config.retraining.sharpe_floor})")
   - Otherwise return (False, "")

3. **`check_data_freshness(db, config, env_name) -> bool`**:
   - For equity: query MAX(date) from ohlcv_daily, check < config.retraining.equity_freshness_days calendar days old
   - For crypto: query MAX(datetime) from ohlcv_4h, check < config.retraining.crypto_freshness_hours hours old
   - Return True if fresh, False if stale

4. **`_update_retrain_state(db, env_name, *, success: bool, failure_reason: str | None = None, generation_increment: bool = False)`**:
   - Upsert into retrain_state table
   - On success: set last_retrain_at to UTC now, reset consecutive_failures to 0
   - On crash failure: increment consecutive_failures, set last_failure_reason
   - If consecutive_failures >= config.retraining.max_consecutive_failures: set enabled=0
   - If generation_increment: increment current_generation by 1

5. **`_ingest_retrain_results(memory_client, env_name, results, trigger_reason, algo_metrics)`**:
   - Ingest per-algo results with source tag `training_run:retrain`
   - Ingest combined ensemble summary
   - Include trigger_reason in metadata
   - Fail-open: wrap in try/except, log warning on failure

6. **`run_retrain(config_path: str, env_name: str, trigger_reason: str, dry_run: bool = False, scheduler_mode: bool = False, timesteps_override: int | None = None) -> int`**:
   - This is the main entry point called by the retrain subprocess
   - Load config via load_config(config_path)
   - Initialize DatabaseManager, check retrain_state.enabled. If disabled, log and return 5
   - Check data freshness. If stale, attempt incremental ingest (import and call ingest commands). If still stale after ingest, return 4
   - Based on config.retraining.use_memory_agent:
     - True: use MetaTrainingOrchestrator with trigger_reason in run_config request
     - False: use train_pipeline.py logic directly (baseline mode)
   - Run all 3 algos (PPO, A2C, SAC) sequentially for env_name. Track per-algo duration
   - Run walk-forward validation via WalkForwardBacktester
   - Check ensemble gate via check_ensemble_gate()
   - If gate fails: _update_retrain_state(success=False), ingest results, consolidate memory, return 1
   - If gate passes and not dry_run: deploy_to_shadow(), update shadow_deployed_at, increment generation
   - _update_retrain_state(success=True, generation_increment=True)
   - Ingest retrain results with ':retrain' source tags
   - Always call MemoryClient.consolidate() (success or failure)
   - Track total_duration_sec plus per-algo durations, store in DuckDB training_runs with run_type and trigger_reason
   - Return 0 on success

7. **`spawn_retrain_subprocess(env_name, config_path, trigger_reason, dry_run=False, timesteps_override=None) -> int`**:
   - Construct command: `[sys.executable, "scripts/retrain.py", "--env", env_name, "--scheduler-mode", "--trigger-reason", trigger_reason]`
   - If dry_run: append `--dry-run`
   - If timesteps_override: append `--timesteps`, str(timesteps_override)
   - Spawn via subprocess.Popen with `preexec_fn=lambda: os.nice(10)`, stderr=subprocess.PIPE
   - Call proc.wait() to block until completion
   - Return exit code

Exit code convention:
- 0: success (gate passed, deployed to shadow)
- 1: gate failed (trained OK but Sharpe/MDD below threshold)
- 2: crash/exception
- 3: timeout
- 4: data freshness check failed
- 5: retrain disabled for this environment

   - Rewrite `duckdb(self)` context manager to:
     ```python
     @contextlib.contextmanager
     def duckdb(self) -> Generator[Any, None, None]:
         """Open fresh DuckDB connection, yield, close. No persistent state.

         Lock is held only for the duration of the context block (milliseconds
         for reads, seconds for writes). Enables concurrent subprocess access.
         """
         self._duckdb_path.parent.mkdir(parents=True, exist_ok=True)
         conn = duckdb.connect(str(self._duckdb_path))
         try:
             yield conn
         finally:
             conn.close()
     ```
   - Update `close(self)` to remove the duckdb_conn cleanup block (lines 507-514). Keep only the method signature and a log.info("database_manager_closed") call
   - Update `reset(cls)` classmethod: remove the `cls._instance.close()` call that tries to close persistent conn. Instead just set `cls._instance = None`
   - Update docstring: change "DuckDB uses a persistent connection with per-call cursors" to "DuckDB opens a fresh connection per context manager call (short-lived)"

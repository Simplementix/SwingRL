    ```python
    class Alerter:
        """Discord webhook alerter with level-based routing and cooldown."""

        def __init__(
            self,
            webhook_url: str,
            cooldown_minutes: int = 30,
            consecutive_failures_before_alert: int = 3,
        ) -> None:
            """Initialize alerter.

            Args:
                webhook_url: Discord webhook URL. Empty string disables sending.
                cooldown_minutes: Minimum minutes between identical critical/warning alerts.
                consecutive_failures_before_alert: Number of consecutive failures before alerting.
            """
    ```

    Implementation details:
    - `_webhook_url: str` — if empty, all sends become no-ops (log only)
    - `_cooldown_minutes: int` — from config
    - `_info_buffer: deque[dict]` — thread-safe buffer for info-level alerts
    - `_lock: threading.Lock` — protects _info_buffer and _last_alert_times
    - `_last_alert_times: dict[str, datetime]` — keyed by "level:title" for cooldown tracking
    - `_failure_counts: dict[str, int]` — keyed by "level:title" for consecutive failure tracking
    - `_seen_hashes: set[str]` — message_hash dedup within digest window

    **send_alert(level, title, message, environment=None):**
    - Validate level is "critical" | "warning" | "info"
    - If level == "info": hash title+message, add to _info_buffer if not duplicate, return
    - For critical/warning: check cooldown (skip if within window), check consecutive failures threshold
    - Build embed payload with color-coded sidebar: critical=0xFF0000, warning=0xFFA500, info=0x3498DB
    - Include: title, description=message, color, timestamp (UTC ISO-8601), footer text "SwingRL | {LEVEL}"
    - If environment provided, prepend to footer: "SwingRL | Equity | CRITICAL"
    - POST via httpx with timeout=10.0
    - Wrap in try/except — log failure with structlog, never raise
    - Update _last_alert_times on successful send

    **send_daily_digest():**
    - Acquire lock, drain _info_buffer into local list, clear _seen_hashes
    - If empty, return
    - Build single embed with title "Daily Digest", description = bulleted list of buffered items
    - Color = 0x3498DB (info blue)
    - POST via httpx

    **_build_embed(level, title, message, environment=None) -> dict:**
    - Returns the Discord webhook JSON payload (embeds array)

    **_post_webhook(payload) -> bool:**
    - httpx.post(self._webhook_url, json=payload, timeout=10.0)
    - response.raise_for_status()
    - Return True on success, False on failure (caught exception)

    - Verify JSON payloads sent to httpx.post match expected embed structure
    - Test cooldown by patching datetime.now(UTC) to control time
    - Test thread safety with ThreadPoolExecutor(4) sending concurrent alerts
    - Test empty webhook_url mode (dev/test)
    - No SQLite dependency in tests (alert_log integration is Plan 01's domain — alerter works standalone)

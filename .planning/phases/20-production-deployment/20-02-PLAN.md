       - `query_position_advice(env, signals, context)` -> dict: System prompt focuses on position-level insights from memory. Returns {"advice": str, "rationale": str}. Advisory only.
       - `query_trade_veto(env, signal, context)` -> dict: System prompt focuses on whether a specific trade should be blocked based on memory patterns. Returns {"vetoed": bool, "rationale": str}.

    3. Each method: pull recent consolidations + memories via `get_consolidations()` and `get_memories()`, XML-wrap them into the prompt, call Ollama with structured JSON format, parse response, clamp values, return safe default on any exception.

    4. Use httpx.AsyncClient (consistent with QueryAgent pattern in same service).

    5. Write tests in `tests/memory_service/test_live_agent.py` with mocked httpx responses.
